## Change log

Implemented input handling for dish name, description, course, and price, including automatic generation of unique IDs for each entry.

Added success confirmation upon saving new dishes.

Applied input trimming to remove leading and trailing whitespace.

Introduced validation to prevent submission of zero or negative price values.

Added duplicate-entry protection to ensure the same dish and course cannot be added more than once.

Ensured consistent look and feel across all screens.

Standardised heading styles for clarity and hierarchy.

Improved button visibility to make key actions easy to identify.

Applied a unified, pleasant colour theme throughout the interface.

Refined spacing to maintain uniform separation between sections.

Implemented gentle inline confirmation and error messages near input fields, in addition to pop-up alerts, to support user focus and understanding.
